# مصفوفة تحقق لوحة الطالب

هذه المصفوفة تفرق بين "الشكل موجود" و"المنطق يعمل" حتى لا يتم اعتبار صفحة جميلة كأنها مكتملة بدون اختبار وظيفي.

| الجزء | مصدر البيانات/المنطق | الحالة الحالية | التحقق المطلوب |
| --- | --- | --- | --- |
| الرئيسية | الكورسات، الدروس المكتملة، نتائج الاختبارات، المهارات الضعيفة | موجودة ومربوطة بالبيانات | الدخول كطالب والتأكد من الأرقام والتقدم |
| دوراتي | `AccessGrant` والكورسات المجانية | موجودة | فتح كورس والتأكد من روابط الدروس |
| التعلم الذكي | `SkillProgress` ثم دروس/اختبارات مرتبطة بالمهارة | تم تصحيح البحث عن الاختبارات عبر الأسئلة | طالب لديه مهارات ضعيفة يرى توصيات حقيقية |
| جلساتي | `ActivityLog` للحجز وحقول اجتماعات الدروس | موجودة | حجز جلسة والتأكد من ظهورها |
| مركز ساهر | بنك الأسئلة النشط + إنشاء Quiz من نوع `saher` | تم تصحيح الأعطال الحرجة | توليد اختبار، بدء المحاولة، تسليمها |
| سجل اختباراتي | `QuizAttempt` و`QuizResult` | موجود | طالب له محاولات يرى التجميع والنتائج |
| مركز مراجعة الأسئلة | `Favorite` و`ReviewLater` والأسئلة الخاطئة من المحاولات | موجود | إضافة سؤال للمفضلة/راجع لاحقا ثم مراجعته |
| التقارير | بيانات تقدم الطالب داخل الواجهة | شكل موجود | مطابقة التقرير مع بيانات فعلية لطالب لديه نشاط |
| خطتي | Alpine/localStorage داخل الصفحة | شكل ومنطق واجهة موجود | إضافة/حذف مهمة والتأكد من حفظها في المتصفح |
| المدفوعات | `PaymentRequest` للطالب | موجود | طلب شراء/رفع إيصال ثم متابعة الحالة |
| صفحات التصنيف العامة | `Path`, `Subject`, `Skill` عبر HasManyThrough | تم إصلاح استعلامات المهارات | فتح `/category/1` و`/category/1/subject/1` بعد النشر |
| الحماية | Middleware `auth` + `role:student` | موجود | غير الطالب لا يدخل لوحة الطالب |

## قرار الجاهزية

- جاهز للاختبار المرحلي بعد push وتشغيل migrations.
- ليس جاهزا لوصفه "منتهيا بالكامل" قبل اختبار دخول طالب وتجربة ساهر والمدفوعات والملفات المرفوعة على Hostinger.
